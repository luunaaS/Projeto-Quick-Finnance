# Documentação do Projeto Quick Finnance

Esta pasta contém toda a documentação do sistema Quick Finnance, incluindo manuais técnicos, guias do usuário e relatórios de testes.

## Documentos Disponíveis

### 1. [Documentação Técnica](DOCUMENTATION.md)
Visão geral da arquitetura do sistema, tecnologias utilizadas, estrutura de módulos e instruções de instalação e execução.

**Conteúdo**:
- Visão geral do projeto
- Arquitetura do sistema (Backend e Frontend)
- Tecnologias utilizadas
- Esquema do banco de dados
- Endpoints da API (resumo)
- Instruções de instalação e execução

### 2. [Manual do Usuário](USER_MANUAL.md)
Guia completo para os usuários finais do sistema, com instruções passo a passo sobre como utilizar todas as funcionalidades.

**Conteúdo**:
- Primeiros passos (criação de conta e login)
- Conhecendo o Dashboard
- Gerenciamento de transações
- Análise de relatórios
- Definição de metas financeiras
- Configurações do sistema

### 3. [Documentação da API](API_DOCUMENTATION.md)
Referência completa dos endpoints da API REST, incluindo exemplos de requisições e respostas.

**Conteúdo**:
- Introdução à API
- Autenticação (JWT)
- Endpoints de autenticação
- Endpoints de transações
- Endpoints de categorias
- Endpoints de financiamentos
- Endpoints de metas
- Endpoints de relatórios
- Códigos de status HTTP
- Exemplos de uso (cURL e JavaScript)

### 4. [Relatório de Testes de Validação](TEST_VALIDATION_REPORT.md)
Relatório detalhado dos testes executados no sistema, incluindo configuração do ambiente, testes implementados e resultados.

**Conteúdo**:
- Resumo executivo
- Configuração do ambiente de testes
- Testes implementados
- Resultados da execução
- Cobertura de testes
- Recomendações

## Como Usar Esta Documentação

- **Desenvolvedores**: Consulte a Documentação Técnica e a Documentação da API para entender a arquitetura e integrar com o sistema.
- **Usuários Finais**: Leia o Manual do Usuário para aprender a usar o sistema de forma eficiente.
- **Gerentes de Projeto**: Revise as Especificações Funcionais e o Relatório de Testes para acompanhar o progresso e a qualidade do projeto.
